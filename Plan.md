# План автоматизации тестирования

## 1. Перечень автоматизируемых сценариев

### Позитивные сценарии:
- Успешная оплата по карте (APPROVED)
- Успешная покупка в кредит (APPROVED)

### Негативные сценарии:
- Оплата отклонённой картой (DECLINED)
- Неверный номер карты (менее 16 цифр)
- Неверный месяц (00, 13)
- Неверный год (прошедший)
- Пустые поля формы
- Неверное имя владельца (кириллица, цифры)
- Неверный CVC (менее 3 цифр)

---

## 2. Инструменты

- **Java** — основной язык для написания автотестов
- **JUnit 5** — фреймворк для организации и запуска тестов
- **Selenide** — библиотека для UI-тестирования (упрощает работу с Selenium)
- **Gradle** — система сборки проекта
- **Docker** — для поднятия тестового окружения (БД и банковского сервиса)
- **Allure** — для генерации отчётов по тестированию

---

## 3. Риски

- Нестабильная работа UI (изменение локаторов)
- Ошибки при запуске Docker
- Несовместимость версий браузера и драйвера
- Зависимость тестов от состояния базы данных
- Падение тестов из-за сетевых задержек

---

## 4. Оценка трудозатрат (в часах)

- Настройка окружения — 2 часа
- Написание тестов — 6 часов
- Отладка и исправление ошибок — 4 часа
- Подготовка отчётов — 2 часа

**Итого: ~14 часов**

---

## 5. План сдачи работ

- День 1–2: настройка окружения и написание тестов
- День 3: прогон тестов и исправление ошибок
- День 4: подготовка отчётов и финальная проверка
